# 개발자 온보딩 간소화 가이드

## 개요
새로운 개발자가 장비 관리 시스템 프로젝트에 빠르게 적응하고 생산성을 발휘할 수 있도록 돕는 온보딩 단계별 가이드입니다. 이 문서는 기존의 포괄적인 개발 가이드라인을 보완하며, 처음 프로젝트에 합류하는 개발자들을 위한 단계적 접근법을 제시합니다.

## 시작하기 전에
- 개발 환경에 Node.js(v18 이상), pnpm, Git이 설치되어 있는지 확인하세요.
- Azure 계정과 필요한 액세스 권한이 부여되었는지 확인하세요.
- VS Code와 권장 확장 프로그램을 설치하세요.

## 1단계: 환경 설정 (1일차)

### 코드 받기 및 환경 설정
```bash
# 레포지토리 클론
git clone https://github.com/your-org/equipment-management-system.git

# 프로젝트 폴더로 이동
cd equipment-management-system

# 의존성 설치
pnpm install

# 환경 변수 설정
cp .env.example .env.local
```

### 환경 변수 설정
`.env.local` 파일을 열고 다음 필수 변수들을 채워넣으세요:
- `AZURE_AD_CLIENT_ID`
- `AZURE_AD_CLIENT_SECRET`
- `AZURE_AD_TENANT_ID`
- `NEXTAUTH_SECRET`
- `NEXT_PUBLIC_API_URL`

### 첫 실행
```bash
# 개발 모드로 전체 앱 실행
pnpm dev

# 또는 특정 앱만 실행
pnpm dev:frontend
pnpm dev:backend
```

## 2단계: 프로젝트 탐색 (1-2일차)

### 핵심 폴더 구조 이해
```
equipment-management-system/
├── apps/
│   ├── frontend/ (Next.js 앱)
│   └── backend/ (NestJS 앱)
├── packages/ (공유 코드)
└── docs/ (프로젝트 문서)
```

### 주요 문서 읽기
1. `README.md` - 프로젝트 개요
2. `docs/architecture/` - 전체 아키텍처 이해
3. `docs/guides/onboarding.md` - 상세 온보딩 가이드

### 핵심 기능 파악
1. 장비 관리 (조회, 등록, 업데이트, 삭제)
2. 대여 관리 (요청, 승인, 반납)
3. 사용자 관리 및 권한 체계

3. 모델 훅: `packages/api-client/src/hooks/use-equipment-api.ts`

## 자주 묻는 질문 (FAQ)

**Q: 개발 환경에서 인증을 어떻게 테스트하나요?**  
A: 개발 환경에서는 `apps/backend/src/auth/mocks/`에 있는 테스트 계정을 사용할 수 있습니다.

**Q: 새로운 컴포넌트를 어디에 추가해야 하나요?**
A: `apps/frontend/src/components/` 아래에 도메인별로 구성된 폴더에 추가하세요. 공유 타입이나 스키마는 `packages/schemas/`에 추가합니다.

**Q: API 엔드포인트를 어떻게 테스트하나요?**  
A: `rest-client` VS Code 확장을 설치하고 `docs/api-examples/`의 예제를 참조하세요.

**Q: 모노레포에서 새 패키지를 추가하려면 어떻게 해야 하나요?**  
A: `pnpm create turbo@latest -y`를 실행하고 지침을 따르세요. 팀 리드에게 새 패키지 추가 계획을 먼저 공유하는 것이 좋습니다.
