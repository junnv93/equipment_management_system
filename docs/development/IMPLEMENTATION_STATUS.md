# 구현 상태 분석 및 미구현 기능 정리

현재 프로젝트의 구현 상태와 미구현 기능을 정리한 문서입니다.

---

## 현재 구현 상태

### 완전 구현

1. **기본 인증 시스템**

   - Azure AD 통합
   - JWT 기반 인증
   - 기본 역할: USER, MANAGER, ADMIN
   - 권한 기반 접근 제어 (RBAC)

2. **장비 관리 (기본 CRUD)**

   - 장비 등록/수정/삭제 (직접 처리, 승인 프로세스 없음)
   - 검색 및 필터링
   - 상태 관리
   - 기본 필드만 포함

3. **대여 관리**

   - 대여 신청/승인/반려
   - 1단계 승인 프로세스
   - 반려 사유 필수
   - 장비 소유 팀의 담당자/매니저 승인

4. **반출 관리 (기본)**

   - 내부 목적 반출 (1단계 승인)
   - 외부 대여 목적 반출 (2단계 승인)
   - 기본 반입 처리 (검사 항목 포함)

5. **교정 관리 (기본)**
   - 교정 기록 등록 (승인 프로세스 없음)
   - 교정 일정 조회
   - 교정 알림

### 부분 구현

1. **반입 프로세스**

   - 검사 항목 기록 가능
   - 기술책임자 승인 없음

2. **알림 시스템**
   - 교정 알림 있음
   - 중간점검 알림 없음

## 미구현 기능 목록

### Phase 1: 핵심 인프라 (최우선)

1. **사용자 역할 시스템 개선**

   - 역할 변경: USER → 시험실무자, MANAGER → 기술책임자, ADMIN → 시험소별 관리자
   - 사이트 정보 추가 (수원랩, 의왕랩)
   - Azure AD 팀 정보 자동 매핑

2. **사이트별 권한 관리**
   - 장비 등록 시 사이트 지정
   - 사이트별 조회 권한
   - 팀별 권한 제한

### Phase 2: 승인 프로세스 (고우선)

3. **장비 등록/수정/삭제 승인 프로세스** - ✅ **완료** (2026-01-19)

   - ✅ 2단계 승인 (요청 → 기술책임자 승인)
   - ✅ 파일 첨부 (이력카드, 검수보고서)
   - ✅ 필수 필드 확장
   - ✅ E2E 테스트 완료 (13/14 통과)
   - 관련 문서: `EQUIPMENT_APPROVAL_TEST_RESULTS.md`

4. **교정 관리 승인 프로세스**

   - 기술책임자 등록: Comment 입력
   - 시험실무자 등록: 기술책임자 승인 필요
   - 중간점검 알림

5. **반입 프로세스 개선**
   - 반입 시 검사 항목
   - 기술책임자 승인

### Phase 3: 고급 기능 (중우선)

6. **보정계수 관리**

   - 보정계수 등록/수정
   - 기술책임자 승인
   - 보정계수 대장

7. **부적합 장비 관리**

   - 부적합 발견 시 사용 중단
   - 원인분석/조치 기록
   - 사용 재개 승인

8. **중간점검 알림**
   - 중간점검일정 기반 알림
   - 교정 알림과 통합

### Phase 4: 관리 기능 (중우선)

9. **공용장비 관리**

   - Safety lab 장비 등록
   - 일회성 교정성적서/관리기록 등록

10. **소프트웨어 관리대장**

    - Software 등록/변경
    - 검증 기록 보관
    - Software 변경 History
    - Software 통합 관리대장

11. **교정계획서**
    - 연간 교정계획서 작성
    - 시험소장 승인 프로세스
    - 계획서 출력 (PDF)

### Phase 5: 데이터 관리 (저우선)

12. **데이터 보관 정책**

    - 5년 보관 정책
    - 용량 관리

13. **데이터 백업**

    - 백업 주기 설정
    - 백업본 관리

14. **감사 로그 시스템**
    - 모든 변경 사항 자동 기록
    - 로그 조회 기능
