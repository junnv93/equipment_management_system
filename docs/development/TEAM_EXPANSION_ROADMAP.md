```

현재 (1인)      →  확장 준비 (1-2인)  →  팀 운영 (3인+)
로컬 개발          하이브리드            완전 컨테이너화
┌─────────┐       ┌──────────┐         ┌───────────┐
│ 로컬    │       │ Docker + │         │ Docker    │
```

## 📅 Phase 2: 팀 확장 준비 (1-2인)

**기간:** 첫 팀원 합류 2개월 전 ~ 합류 직전
**목표:** 온보딩 자동화, 개발 환경 표준화

### ✅ 개발 환경 (하이브리드)

#### 옵션 A: Volume Mount 방식 (추천)
```yaml
# docker-compose.dev.yml
services:
  backend:
    build:
      context: .
      target: development
    volumes:
      - ./apps/backend:/app/apps/backend
      - ./packages:/app/packages
      - /app/node_modules
    command: pnpm --filter backend run dev
    ports:
      - "3001:3001"
```

#### 옵션 B: 계속 로컬 (간단)
```bash
# 변화 없음 - 문서화와 스크립트만 개선
docker-compose up -d postgres redis
pnpm dev
```

## 🎯 핵심 원칙

1. **점진적 개선:** 한 번에 모든 것을 바꾸지 않는다
2. **실용주의:** 지금 당장 필요한 것만 한다
3. **문서화 우선:** 코드보다 문서를 먼저 업데이트
4. **자동화:** 반복 작업은 스크립트로
5. **검증:** 각 Phase 전환 전 반드시 테스트
