# 시험 소프트웨어의 유효성확인 (UL-QP-18-09)

> **양식번호**: UL-QP-18-09  
> **관련 절차서**: UL-QP-18 장비 관리 절차서, 14. 소프트웨어의 유효성 확인

이 문서는 시험 소프트웨어의 유효성 확인 양식의 구조와 작성 가이드를 정의한다.

---

## 양식 개요

시험 소프트웨어의 유효성 확인은 **두 가지 방법** 중 하나로 수행된다:

| 방법                     | 설명                                            | 적용 조건                                             |
| ------------------------ | ----------------------------------------------- | ----------------------------------------------------- |
| **방법 1: 공급자 시연**  | 공급자가 소프트웨어에 대한 유효성을 검증한 경우 | 공급자의 검증 기록을 수령하여 대체                    |
| **방법 2: UL 자체 시험** | UL이 직접 유효성확인 시험을 실시                | 공급자가 검증하지 않거나, 자체 개발 소프트웨어인 경우 |

---

## 방법 2: UL 자체 유효성확인 시험 (Validation Tests by UL)

### 기본 정보

| 항목                       | 영문                                  | 설명                 |
| -------------------------- | ------------------------------------- | -------------------- |
| 소프트웨어명 또는 상세사항 | Software Name or Description          | 소프트웨어 식별 정보 |
| 소프트웨어 제작자          | Software Author                       | 개발자/제작 업체     |
| 소프트웨어 버전/수정일     | Software Version and/or revision date | 버전 및 날짜         |
| 참고 문서                  | Reference documents                   | 관련 참조 문서       |
| 조작 단위 설명             | Operating unit description            | 운용 환경 설명       |
| 소프트웨어 구성            | Software components                   | SW 구성 요소         |
| 하드웨어 구성              | Hardware components                   | HW 구성 요소         |

### 검증 항목

#### 1) 획득 기능 (Acquisition Functions)

| 항목               | 설명                                          |
| ------------------ | --------------------------------------------- |
| 기능 명칭/상세사항 | 검증할 데이터 획득 기능의 이름과 설명         |
| 독립 획득 방법     | 소프트웨어와 별개로 시험 데이터를 획득한 방법 |
| 수락 기준          | 합격/불합격 판정 기준                         |
| 첨부               | 소프트웨어/독립 방법으로 획득한 데이터 비교   |

> **참고**: 획득 기능이 사용자 입력에 따라 달라지는 경우, 수차례 충분히 시험을 반복

#### 2) 프로세싱 기능 (Processing Functions)

| 항목               | 설명                                      |
| ------------------ | ----------------------------------------- |
| 기능 명칭/상세사항 | 검증할 데이터 처리 기능의 이름과 설명     |
| 독립 처리 방법     | 소프트웨어와 별개로 데이터를 처리한 방법  |
| 수락 기준          | 합격/불합격 판정 기준                     |
| 첨부               | 입력 데이터, SW 출력, 독립 방법 출력 비교 |

#### 3) 제어 기능 (Control Functions)

| 항목               | 설명                                               |
| ------------------ | -------------------------------------------------- |
| 제어된 장비 기능   | Equipment function controlled                      |
| 예상되는 장비 기능 | Expected equipment function                        |
| 확인되는 장비 기능 | Observed equipment function                        |
| 독립 판정 방법     | 제어 소프트웨어와 무관하게 장비 기능을 판정한 방법 |
| 수락 기준          | 합격/불합격 판정 기준                              |
| 첨부               | 로우 데이터, 확인사항, 관련 정보                   |

> **참고**: 제어 기능이 사용자 입력에 따라 달라지는 경우, 수차례 충분히 시험을 반복

## 워크플로우

```
1. 소프트웨어 도입 요청
   └─ 기술책임자가 유효성 확인 필요 여부 판단

2. 유효성 확인 방법 결정
   ├─ 공급자 검증 기록 있음 → 방법 1 (공급자 시연)
   └─ 공급자 검증 없음/자체 개발 → 방법 2 (UL 자체 시험)

3. 유효성 확인 실시
   └─ UL-QP-18-09 양식 작성

4. 결과 제출
   └─ 기술책임자 → 품질책임자에게 결과 제출

5. 소프트웨어 등록
   └─ 품질책임자가 시험용 소프트웨어 관리대장(UL-QP-18-07)에 등록
```
