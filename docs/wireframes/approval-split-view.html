Design Decisions

  1. Split View vs Modal: 리스트에서 항목 클릭 시 우측 패널에 즉시 렌더링. 모달처럼 컨텍스트가 차단되지 않아 연속 triage가 빠름.
  2. Auto-Advance: 승인/반려 후 자동으로 다음 항목 포커스. Gmail의 auto-advance와 동일 패턴. onSuccessCallback에서 다음 인덱스 선택.
  3. 키보드 Triage: ↑↓ 이동, A 승인, R 반려, Space 선택. 파워 유저가 마우스 없이 20+건 연속 처리 가능.
  4. Active Row Indicator: 선택된 행에 좌측 2px 블루 보더 + 연한 블루 배경. 시각적으로 리스트↔디테일 연결 명확.
  5. Sticky Footer: 상세 패널의 승인/반려 버튼이 항상 하단 고정. 스크롤 위치와 무관하게 즉시 액션 가능.
  6. 모바일 Fallback: lg: 미만에서는 기존 Modal 패턴 유지. Split View는 min-width: 1280px 이상에서만 활성화.
  7. Design Token 호환: 기존 APPROVAL_ROW_TOKENS, APPROVAL_KPI_STRIP_TOKENS 등 모든 토큰이 그대로 재사용됨. 새 토큰은 Detail Panel 전용 3개만 추가.
  8. Row 인라인 액션: 호버 시 ✓/✕ 버튼 강조. 상세 확인 불필요한 명확한 건은 리스트에서 바로 처리 가능.
