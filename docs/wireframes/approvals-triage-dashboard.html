디자인 개선 포인트 — 아키텍처 변경 사항

1 탭 → 사이드바 카테고리 내비게이션

  • 기술책임자 8탭 2줄 오버플로우 → 섹션 그룹 사이드바로 전환
  • 각 카테고리의 대기 건수를 항상 볼 수 있어 전체 상황 파악 용이
  • 긴급 건(urgent count)은 빨간 뱃지로 시각적 경고
  • SSOT: 기존 ROLE_TABS, TAB_META 그대로 활용

2 카드 → 컴팩트 테이블 로우

  • 카드당 ~150px → 로우당 ~64px (약 60% 공간 절약)
  • 한 화면에 8-12건 표시 가능 (기존 3-4건)
  • 경과일 urgency가 좌측 보더+배경색으로 즉시 식별
  • content-visibility: auto 유지 (대량 목록 성능)

3 KPI 스트립 추가

  • 전체 대기, 긴급 건수, 평균 대기일, 금일 처리 현황 한눈에
  • 관리자 의사결정 맥락(context) 제공
  • 기존 queryKeys.approvals.counts 쿼리 확장 활용
  • KPI → 카테고리 클릭으로 필터링 내비게이션

4 SSOT 호환 — 기존 아키텍처 유지

  • TAB_META에 multiStep, section 플래그 추가
  • REJECTION_MIN_LENGTH = 10 상수화
  • Design Token Layer 3에 APPROVAL_ROW_TOKENS 추가
  • 다단계 승인: 인라인 step-badge (기존 StepIndicator는 상세 모달 전용)
  • 기존 approvals-api.ts 1,337줄 전혀 변경 없음

현재 구조 vs 제안 구조 비교

현재 (Card + Tabs)
레이아웃: 탭 2줄 → 카드 리스트 (세로 스크롤)
정보 밀도: 카드당 ~150px, 화면 3-4건
Urgency: 텍스트 색상만 변경 (놓치기 쉬움)
전체 현황: 탭 뱃지 숫자만 (맥락 부족)
다단계: 카드 내 StepIndicator 전체 표시
isMultiStep: 3곳 하드코딩 카테고리 체크
반려 최소길이: 매직넘버 10 × 3곳
→
제안 (Row + Sidebar + KPI)
레이아웃: 사이드바 섹션 + 컴팩트 로우
정보 밀도: 로우당 ~64px, 화면 8-12건
Urgency: 좌측 보더 + 배경색 + 볼드 (즉시 식별)
전체 현황: KPI 4개 + 사이드바 카운트
다단계: 인라인 dot-badge (상세 모달에서 전체)
isMultiStep: TAB_META.multiStep 플래그
반려 최소길이: REJECTION_MIN_LENGTH 상수
