⬡ 설계 주석 (Design Annotations)
📐 Information Architecture
1st axis: Site (수원/의왕/평택)
사이트당 팀이 1개뿐인 도메인 모델에 맞게 사이트를 primary grouping으로 변경.
분류 필터 pill은 2nd axis로 보조 역할.
🎨 Visual Language
사이트: UL 팔레트 3색 identity (Midnight/Blue/Green)
분류: classification code letter (E/R/W/S/A/P) + 고유 색상
hover 시 left accent bar 3px 나타남 → classification 색상
🔑 Key Interactions
팀 행 클릭 → 우측 슬라이드 detail panel (팀원 + 정보)
사이트 패널 내 "팀 추가" → 해당 사이트 pre-filled 생성 폼
상단 "팀 추가" → 사이트 선택 먼저
🔒 권한별 렌더링
technical_manager: 자기 사이트 1개 컬럼만 표시
lab_manager: 전체 3컬럼 + 팀 추가 버튼 표시
system_admin: 모든 권한 + 삭제 버튼 표시
📦 컴포넌트 구조
TeamListContent → SiteColumn (×N) → TeamRow
Detail Panel: 오른쪽 슬라이드 패널 (역할 그룹별 팀원 목록)
필터는 URL SSOT (useTeamFilters) 유지
💡 이전 v1과의 차이
v1: 분류별 섹션 + 좌측 필터 패널 (heavy)
v2: 사이트별 컬럼 + 상단 필터 pill (light)
팀 수가 적을 때 compact row가 card보다 밀도 효율 높음
