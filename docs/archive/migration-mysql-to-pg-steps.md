# MySQL에서 PostgreSQL로 마이그레이션 단계

이 문서는 장비 관리 시스템의 데이터베이스를 MySQL에서 PostgreSQL로 마이그레이션하는 단계를 설명합니다.

## 준비 사항

1. MySQL 서버와 PostgreSQL 서버가 설치되어 있어야 합니다.
2. 환경 변수 설정이 완료되어 있어야 합니다 (.env 파일 참조).

## 단계별 마이그레이션 가이드

### 1. MySQL 데이터베이스 준비

MySQL 데이터베이스에 초기 데이터가 있는지 확인합니다. 없다면 다음 SQL 스크립트를 사용하여 샘플 데이터를 생성합니다:

```sql
-- init-mysql.sql 파일 참조
```

### 2. PostgreSQL 데이터베이스 생성

PostgreSQL 서버에 데이터베이스를 생성합니다:

```sql
CREATE DATABASE equipment_management;
```

### 3. PostgreSQL 스키마 생성

다음 명령어를 사용하여 PostgreSQL에 스키마를 생성합니다:

```sql
-- apps/backend/drizzle/0000_initial_migration.sql 파일 참조
```

### 4. 데이터 마이그레이션 실행

마이그레이션 스크립트를 실행하여 데이터를 이전합니다:

```bash
pnpm run migrate
```

### 5. 백엔드 애플리케이션 설정 업데이트

백엔드 애플리케이션의 설정을 PostgreSQL을 사용하도록 업데이트합니다:

1. `apps/backend/.env` 파일에서 데이터베이스 설정을 PostgreSQL로 변경합니다.
2. 백엔드 서버를 재시작합니다.

### 6. 검증

마이그레이션이 성공적으로 완료되었는지 검증합니다:

1. PostgreSQL 데이터베이스의 데이터 확인
2. 백엔드 애플리케이션이 PostgreSQL과 연결되어 정상 작동하는지 확인

## 문제 해결

마이그레이션 도중 문제가 발생할 경우 다음 사항을 확인하세요:

1. MySQL과 PostgreSQL 서버가 실행 중인지 확인
2. 환경 변수가 올바르게 설정되었는지 확인
3. 필요한 모든 의존성이 설치되어 있는지 확인
4. 로그를 확인하여 오류 메시지 확인

## 롤백 계획

마이그레이션이 실패하면 다음 롤백 단계를 수행하세요:

1. 백엔드 애플리케이션을 MySQL로 다시 설정
2. PostgreSQL 데이터베이스 삭제
3. MySQL 데이터베이스 복원 (필요한 경우)

## 추가 참고 사항

- 마이그레이션 전 MySQL 데이터베이스 백업을 권장합니다.
- 대규모 데이터베이스의 경우 마이그레이션에 시간이 소요될 수 있습니다.
- 운영 환경에서는 서비스 다운타임 계획을 수립하고 진행하세요.
