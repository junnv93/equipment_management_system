Project in 3 sentences

  1. What: An internal web application that digitalizes UL Solutions Korea's lab equipment management (calibration scheduling, equipment loan/return, multi-stage approvals) per UL Solutions Korea's internal Equipment Management Procedure, used by Suwon / Uiwang / Pyeongtaek labs.
  2. Where it stands: Development is advanced (26 backend modules, full frontend, automated test coverage at unit and end-to-end levels). Cybersecurity Architecture Review with Lance Morrow is in progress; he asked us to move from a desktop PC to a properly managed server before completing the review.
  3. What we need: A properly managed server (on-prem or Azure VM). The ServiceNow link Lance provided returns an authorization error for both myself and Wonsik (local IT) — appears to be a catalog item permission issue.

Why this project (in 4 bullets)

  1. Unified equipment state tracking — loan, calibration, repair, non-conformance, and disposal are all tracked in one record per equipment, replacing fragmented spreadsheets and paper logs across the three labs.
  2. Automated quality document generation — calibration plans, equipment transfer records, inspection reports, and disposal forms are generated from system data as PDF exports rather than filled by hand. Commercial LIMS such as Labware do not include UL Solutions Korea's specific procedure forms out of the box, so equivalent functionality on a commercial product would require similar custom development.
  3. Azure AD–native team mapping & email approval workflows — SSO + automatic site/team assignment from Azure AD groups + email notifications at each approval step.
  4. Daily calibration digest emails — automated digest summarizing equipment with calibration approaching expiry (default 30 days), past expiry, and overdue checkouts. Supports ISO/IEC 17025 §6.4 by making calibration status actionable rather than relying on manual spreadsheet checks.
