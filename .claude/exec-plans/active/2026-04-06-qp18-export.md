# QP-18 양식 내보내기 구현 계획

## 메타
- 생성: 2026-04-06T10:00:00+09:00
- 모드: Mode 2 (Full)
- 예상 변경: ~20개 파일

## 설계 철학
QP-18-01(관리대장) / QP-18-02(이력카드) 양식의 실제 템플릿 파일을 기반으로 데이터를 채워넣는 방식으로 내보내기 구현. 양식 서식을 코드로 재현하지 않음.

## 아키텍처 결정
| 결정 | 선택 | 근거 |
|------|------|------|
| docx 생성 방식 | docxtemplater + 실제 템플릿 | 현재 XML 직접 생성은 양식 충실도 낮음 |
| xlsx 생성 방식 | exceljs + 실제 템플릿 | 서식/머지셀 보존 |
| deputyManagerId 저장 | UUID (users FK) | technicalManager(문자열)와 달리 FK 참조 |
| 서명 이미지 삽입 | docxtemplater-image-module-free | 무료 모듈, base64 이미지 지원 |

## 구현 Phase

**목표:** 장비 등록/수정 폼에서 부담당자 선택, 상세 페이지에서 표시

**변경 파일:**
1. `apps/backend/src/modules/equipment/dto/create-equipment.dto.ts` — 수정: deputyManagerId 추가
2. `apps/backend/src/modules/equipment/dto/update-equipment.dto.ts` — 수정: deputyManagerId 추가
3. `apps/backend/src/modules/equipment/equipment.service.ts` — 수정: create/update/findOne에 deputyManagerId 포함, detail 쿼리에서 name resolve
4. `apps/frontend/components/equipment/StatusLocationSection.tsx` — 수정: 부담당자 select 추가

**검증:** `pnpm tsc --noEmit` 통과, 장비 등록 폼에 부담당자 드롭다운 렌더링
