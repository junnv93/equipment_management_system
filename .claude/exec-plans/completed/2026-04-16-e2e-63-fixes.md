## 설계 철학
E2E 테스트 63건의 실패를 근본 원인별로 분류하고, **서비스(프로덕션 코드)가 올바르면 테스트를 서비스에 맞추고**, 서비스에 비즈니스 로직 누락이 있으면 서비스를 구현한다. **의도적으로 제거된 기능을 되살리지 않는다.**

## 아키텍처 결정 (감사 보정 반영)
| # | 결정 | 선택 | 근거 |
|---|------|------|------|
| 1 | calibrationDue: -1 거부 | Zod `.positive()` → `.int()` 변경 | 서비스가 이미 음수를 "교정 기한 초과"로 처리. 스키마가 의도와 불일치 |
| 2 | repair_history repairedBy/cost/repairCompany | **테스트 기대값 제거** (DB 롤백 금지) | 마이그레이션 `20260204_remove_repair_cost_fields.sql`로 **의도적 삭제됨** |
| 3 | `/equipment/shared` 미구현 | 라우트 신규 구현 | DB에 isShared/sharedSource 컬럼 존재. 비즈니스 요구사항 |
| 4 | CAS version 필수 vs 테스트 미전송 | 테스트에 version 필드 추가 | versionedSchema 필수. CAS 패턴 준수 |
| 5 | non-conformance closedBy body 전송 | 테스트에서 closedBy 제거 (Rule 2) | 서버가 JWT에서 추출. 클라이언트 body 신뢰 금지 |
| 6 | calibration-plans 403 | 테스트에서 `manager` 역할 사용 | CREATE_CALIBRATION_PLAN은 technical_manager 전용 (직무분리) |
| 7 | users profile 403 | 테스트에서 올바른 권한 역할 사용 | test_engineer에 UPDATE_USERS 없음 |
| 8 | equipment-history 배열 vs 객체 | 테스트 단언문을 `response.body.items` 사용으로 수정 | 서버가 `{items, meta}` 반환 |
| 9 | cables measurementDate 형식 | 테스트에서 `YYYY-MM-DD` 사용 | Zod regex가 `/^\d{4}-\d{2}-\d{2}$/` 요구 |
| 10 | audit-logs cursor vs offset | 테스트에 `?page=1` 추가 또는 cursor 응답 단언 수정 | page 미지정 시 cursor 모드 |
| 11 | equipment-filters 가상 상태 | Zod에 없는 status값은 테스트 기대값 수정 | calibration_overdue/retired는 EquipmentStatusEnum 밖 |

## 의사결정 로그 (감사 보정)

1. **repair_history 필드**: ~~DB 마이그레이션으로 추가~~ → **테스트에서 제거**. `20260204_remove_repair_cost_fields.sql` 마이그레이션으로 의도적 삭제됨. 되돌리는 것은 아키텍처 퇴보.

2. **users profile 403**: ~~Zod 400~~ → **권한 403**. test_engineer에 UPDATE_USERS 권한 없음. 올바른 역할 토큰 사용.

3. **equipment-history 배열**: Planner 미탐지. 서버가 `{items, meta}` 페이지네이션 객체 반환, 테스트가 plain array 기대.

4. **cables measurementDate**: Planner 미탐지. ISO datetime vs YYYY-MM-DD regex 불일치.

5. **audit-logs cursor**: Planner 미탐지. page 미지정 시 cursor 모드 응답 (meta 없음).

6. **equipment-filters 가상 상태**: Planner 미탐지. calibration_overdue/retired는 EquipmentStatusEnum에 없음.
